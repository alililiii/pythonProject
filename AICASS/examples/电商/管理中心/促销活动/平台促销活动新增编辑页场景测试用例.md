# 平台促销活动新增/编辑页场景测试用例

## 1. 页面信息

| 项目 | 内容 |
| --- | --- |
| 页面名称 | 平台促销活动新增/编辑页 |
| 页面地址 | `https://ec-hwbeta.casstime.com/admin#/?menuId=platform_promotions&iframeUrl=/admin/merchant#/platform-promotions/add-edit` |
| 登录账号 | `mandy` |
| 所属模块 | 管理中心 / 活动管理 / 平台促销活动管理 |
| 页面形态 | 管理中心主框架内嵌 `iframe` 子应用 |
| 子应用地址 | `/admin/merchant#/platform-promotions/add-edit` |
| 页面用途 | 新增或编辑平台级促销活动，配置活动基础信息、活动对象、促销玩法、活动店铺和活动商品范围，并保存为活动配置。 |

## 2. 页面元素与功能分析

### 2.1 基础信息

| 元素 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 活动名称 | 文本输入框 | 是 | 占位提示为“50个字以内”，用于录入促销活动名称。 |
| 活动时间 | 日期范围控件 | 是 | 包含“开始日期”和“结束日期”，用于控制活动生效时间。 |
| 业务范围 | 单选项 | 是 | 可选“商城业务”“询价业务”。页面提示“选择指定业务后业务范围不可更改”。 |
| 业务线 | 下拉选择框 | 是 | 占位提示为“请选择业务线”，数据来自业务线字典接口。 |

### 2.2 活动对象

| 元素 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 活动区域 | 单选项 | 是 | 可选“不限区域”“按区域设置”。 |
| 活动对象 | 单选项 | 是 | 可选“全部客户”“指定参与客户”“指定不参与客户”。 |

### 2.3 促销玩法

| 元素 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 促销类型 | 单选项 | 是 | 可选“满立折”“满立减”。 |
| 促销条件 | 表格配置 | 是 | 默认展示 1 行配置，支持新增多档规则。 |
| 新增一项配置 | 操作按钮/链接 | 否 | 增加一条促销条件规则。 |
| 营销费用 | 单选项 | 是 | 可选“不设上限”“营销费用上限”。页面提示达到上限后活动自动停止。 |
| 促销叠加规则 | 单选项 | 是 | 可选“不叠加”“叠加”。 |

默认“满立折”促销条件表格字段：

| 字段 | 说明 |
| --- | --- |
| 序号 | 规则序号。 |
| 订单金额 | 满足促销的订单金额门槛。 |
| 折扣 | 折扣值，页面占位提示“折扣范围 0.01 ~ 9.99”。 |
| 平台分摊比例（%） | 平台承担优惠金额比例。 |
| 店铺分摊比例（%）（自动计算） | 根据平台分摊比例自动计算。页面初始值显示为 `0`。 |
| 操作 | 删除或调整规则的操作入口。 |

页面示例文案说明：

> 设置满 100 元打 8.8 折时，客户订单金额为 100，订单实付金额 88，优惠金额 12 元。平台分摊比例 40%，店铺分摊比例 60%。

### 2.4 活动店铺

| 元素 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 活动店铺 | 单选项 | 是 | 可选“不限店铺”“指定可参与店铺”“指定不可参与店铺”。 |

### 2.5 活动商品

| 元素 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 商品范围 | 单选项 | 是 | 可选“按品类配置”“按商品（指定零件号）配置”。 |
| 适用品质 | 单选项 | 否 | 可选“不限品质”“指定品质”。页面提示指定品质后仅展示对应品质的品牌。 |
| 适用配件品牌 | 单选项 | 否 | 可选“不限品牌”“指定品牌”。 |
| 适用品类 | 单选项 | 否 | 可选“不限品类”“指定品类”。 |

### 2.6 页面操作

| 元素 | 类型 | 说明 |
| --- | --- | --- |
| 取消 | 按钮 | 放弃当前新增/编辑内容，返回上一页或列表页。 |
| 保存 | 按钮 | 校验并提交活动配置。 |

## 3. 测试范围

本测试用例聚焦平台促销活动新增/编辑页，覆盖以下能力：

| 测试类别 | 覆盖内容 |
| --- | --- |
| 访问与权限 | 登录态访问、未登录拦截、菜单入口、iframe 加载。 |
| 页面渲染 | 表单分区、字段、默认选项、按钮展示。 |
| 必填校验 | 活动名称、活动时间、业务范围、业务线、活动区域、活动对象、促销类型、促销条件、营销费用、叠加规则、活动店铺、商品范围。 |
| 输入校验 | 名称长度、金额、折扣、分摊比例、时间范围、营销费用上限。 |
| 动态联动 | 单选项切换后的字段展示、表格增删、自动计算。 |
| 保存提交 | 成功保存、失败提示、重复提交、接口异常。 |
| 取消返回 | 编辑未保存时取消、返回列表。 |
| 数据一致性 | 编辑回显、保存后列表/详情一致。 |

## 7. 风险点与注意事项

1. 该页面在管理中心主框架中通过 `iframeCon` 加载 merchant React 子应用，自动化定位时需要进入 iframe 后再定位表单元素。
2. 业务线、品质、客户、品类等选项依赖接口数据，接口异常时页面可能只展示空态。
3. “业务范围”页面提示保存后不可更改，编辑页需重点验证字段锁定和后端校验。
4. 促销条件的“店铺分摊比例”是自动计算字段，应重点验证比例边界值和隐藏字段提交。
5. 切换单选项时，要验证隐藏区域的数据是否被清理，避免提交旧模式下的残留数据。
6. 保存动作可能真实创建测试活动，执行前建议使用带明显前缀的测试数据，并在测试后清理。
