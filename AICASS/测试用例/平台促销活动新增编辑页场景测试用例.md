## 3. 页面元素与规则分析

### 3.1 基础信息

| 元素 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 活动名称 | 文本输入框 | 是 | 新增页必须填写；建议使用唯一名称，例如 `自动化-平台促销-PC-001-YYYYMMDDHHmmss`。 |
| 活动时间 | 日期时间范围控件 | 是 | 开始时间、结束时间均必填；开始时间不得晚于结束时间；建议使用当前时间之后的有效时间段。 |
| 业务范围 | 单选项 | 是 | 可选“商城业务”“询价业务”；保存后业务范围不可更改。Excel 已统一使用“询价业务”，不再使用“询报价业务”。 |
| 业务线 | 下拉选择框 | 是 | 从页面实际业务线选项中选择；空值触发必填校验。 |
| 车辆品牌限制 | 单选项/条件配置 | 是 | 可选“不限车辆品牌”“指定参与车辆品牌”“指定不参与车辆品牌”。选择指定参与或指定不参与时，车辆品牌明细必填。 |
| 车辆品牌明细 | 多选/列表 | 条件必填 | 仅车辆品牌限制为指定参与或指定不参与时填写，多个品牌用分号分隔；示例：`大众;奥迪;斯柯达`。 |

### 3.2 活动对象

| 元素 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 活动区域 | 单选项 | 是 | 可选“不限区域”“按区域设置”。选择“按区域设置”时，活动区域明细必填。 |
| 活动区域明细 | 省市区选择 | 条件必填 | 仅“按区域设置”时填写，建议格式为 `省 > 市 > 区`，例如 `广东省 > 深圳市 > 南山区`。 |
| 活动对象 | 单选项 | 是 | 可选“全部客户”“未交易客户”“已交易客户”“指定可参与客户”“指定不参与客户”。 |
| 客户配置方式 | 单选项/下拉 | 条件必填 | 活动对象为指定可参与或指定不参与时必填，可选“按客户清单配置”“按客户分组配置”。全部客户、未交易客户、已交易客户场景应留空。 |
| 客户清单/分组 | 列表/分组选择 | 条件必填 | 按客户清单配置时选择维修厂客户；按客户分组配置时选择客户分组。 |
| 客户例外 | 例外列表 | 条件必填 | 开启客户例外配置后必须选择例外客户；未开启时应为空。 |

### 3.3 促销玩法

| 元素 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 促销类型 | 单选项 | 是 | 可选“满立减”“满立折”。页面有默认值，负向用例不再构造“促销类型为空”。 |
| 规则条数 | 表格配置 | 是 | 支持一条或多条规则；新增规则后每一档的满额、优惠值和平台分摊均需填写完整。 |
| 订单金额(满额) | 数值输入 | 是 | 必须大于 `0`。多档规则建议按金额递增配置，并验证缺失或重复规则的处理。 |
| 立减/折扣值 | 数值输入 | 是 | 满立减填写立减金额，必须大于 `0`；满立折填写纯数字折扣，例如 `8`、`8.5`、`9.2`，范围应为 `0.01` 到 `9.99`。 |
| 平台分摊值 | 数值输入 | 是 | 满立减填写金额；满立折填写比例数值。满立减场景下平台分摊金额不得超过立减金额；满立折场景下比例不得超过 `100`。 |
| 平台分摊类型 | 规则含义 | 是 | 满立减为“金额”；满立折为“比例(%)”。这是 Excel 修正后的关键差异。 |
| 营销费用 | 单选项 | 是 | 可选“不设上限”“设上限”。选择“设上限”时，营销费用上限必填且必须大于 `0`。 |
| 是否叠加 | 单选项 | 是 | 可选“叠加”“不叠加”。保存后需验证回显。 |

### 3.4 活动店铺

| 元素 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 店铺范围 | 单选项 | 是 | 可选“不限店铺”“指定可参与店铺”“指定不可参与店铺”。 |
| 店铺配置方式 | 单选项/下拉 | 条件必填 | 店铺范围为指定可参与或指定不可参与时必填，可选“按店铺清单配置”“按店铺分组配置”。不限店铺时应留空。 |
| 店铺清单/分组 | 列表/分组选择 | 条件必填 | 按店铺清单配置时选择店铺；按店铺分组配置时选择店铺分组。 |
| 店铺例外 | 例外列表 | 条件必填 | 开启例外店铺配置后必须选择例外店铺；未开启时应为空。 |

### 3.5 活动商品

| 元素 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 商品范围 | 单选项 | 是 | 可选“按品类配置”“按商品(指定零件号)配置”。 |
| 商品限制类型 | 单选项 | 条件必填 | 仅按商品配置时填写，可选“指定促销商品”“指定非促销商品”。 |
| 品质 | 单选项 | 条件必填 | 仅按品类配置时使用，可选“不限品质”“指定品质”。选择指定品质时，品质明细必填。 |
| 配件品牌 | 单选项 | 条件必填 | 仅按品类配置时使用，可选“不限品牌”“指定品牌”。选择指定品牌时，配件品牌明细必填。 |
| 品类 | 单选项 | 条件必填 | 仅按品类配置时使用，可选“不限品类”“指定品类”。选择指定品类时，品类明细必填。 |
| 商品明细 | 商品列表 | 条件必填 | 仅按商品配置时填写；推荐格式：`SKU\|零件号\|商品名\|品牌\|品质\|品类`。按商品配置时，品质、品牌、品类筛选列统一视为“不适用”。 |

### 3.6 页面操作

| 元素 | 类型 | 规则 |
| --- | --- | --- |
| 保存 | 按钮 | 触发表单校验和保存提交；成功后需验证提示、列表可查询和编辑页回显。 |
| 取消 | 按钮 | 放弃当前新增或编辑内容，返回列表页或上一页；未保存数据不得落库。 |

## 10. 风险点与注意事项

1. 真实保存场景会创建平台促销活动，执行前必须确认测试环境允许造数，并准备停用、删除或标记清理策略。
2. 页面在管理中心主框架内通过 `#iframeCon` 加载 merchant 子应用，自动化定位时需要进入 iframe 的 `contentDocument` 后再查找表单元素。
3. 当前给定 `icec-cloud-web-system-manager` 源码只确认管理中心 iframe 加载逻辑，未包含 `/platform-promotions/add-edit` React 子应用源码；字段校验如有线上变更，需要以实际页面和接口响应重新校准。
4. 满立减和满立折的平台分摊含义不同：满立减是金额，满立折是比例。用例、自动化数据和断言必须区分。
5. “全部客户”“未交易客户”“已交易客户”属于自动客群，不应额外提交客户配置方式、客户清单或客户例外。
6. 按商品配置时，品质、品牌、品类筛选不适用，商品信息应下沉到商品明细列表。
7. 切换活动区域、客户、店铺、商品范围后，需要验证隐藏区域数据是否被清理，避免旧数据随保存请求提交。
8. 多档促销规则需要逐档验证满额、优惠值、平台分摊的完整性和边界值。
9. 业务线、区域、客户、店铺、品牌、品类等依赖测试环境接口数据；执行失败时需区分“测试数据不存在”和“页面逻辑错误”。
10. 成功场景保存后至少验证保存成功提示、列表查询、编辑页回显、业务范围锁定；负向场景必须验证未产生活动数据。
